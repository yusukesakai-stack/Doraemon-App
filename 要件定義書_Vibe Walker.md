# 要件定義書_Vibe Walker

## 1. システム概要
Google AI Studio のビルド機能で構築するWebアプリ。ユーザーの現在地と希望散歩時間から、AIが「気持ちいい周回ルート」と「見どころ」を提案する。

## 2. 開発・運用前提
- **デプロイ先**: Google Cloud Run（コンテナ）
- **ソース管理**: GitHub（バックアップ・CI/CD）
- **開発手法**: AI駆動開発（Google AI Studio中心）

## 3. 機能要件
### 3.1 画面構成
- **ホーム画面**: 気分タップ（サクッと15分／ふつう30分／しっかり60分／カスタム）
- **マップ画面**: 生成ルート表示・所要時間・距離・見どころピン
- **詳細モーダル**: 見どころ紹介テキスト・写真
- **再生成ボタン**: 別ルートを提案

### 3.2 機能詳細
| ID | 機能 | 内容 |
|---|---|---|
| F-01 | 現在地取得 | Geolocation APIで位置取得 |
| F-02 | 時間選択 | プリセット＋カスタム入力 |
| F-03 | ルート生成 | Google Maps Directions API + AIスコアリング |
| F-04 | ルート最適化 | 緑・静けさ・安全性を考慮しAIが選定 |
| F-05 | 見どころ抽出 | Places APIから候補→Gemini APIで紹介文生成 |
| F-06 | マップ描画 | Google Maps JavaScript API |
| F-07 | 再生成 | 別候補ルートを提示 |

### 3.3 業務フロー
1. ユーザー来訪→位置情報許可→気分選択
2. ルート候補生成→AIがスコア順に並べ替え→最良案を提示
3. 見どころピン表示→出発→（任意で再生成）

## 4. 非機能要件
- 性能: ルート提案5秒以内
- 可用性: 99.5%
- セキュリティ: HTTPS、位置情報はサーバーに保存しない
- アクセシビリティ: WCAG AA準拠

## 5. 外部システム連携
- Google Maps Platform（Directions / Places / Maps JS）
- Gemini API（ルート評価・紹介文生成）
- Firebase Hosting/Auth（任意）

## 6. データ要件
- 永続データ最小化（履歴は端末ローカル保存）
- ログは個人特定不可な形で集計

## 7. 受け入れ基準
- 指定時間±10%以内で帰着できるルートを生成
- 見どころが最低1件提示される
- スマホで3タップ以内に出発できる

## 8. リスクと対策
- **API利用コスト増** → キャッシュ・利用上限設定
- **位置情報プライバシー** → クライアント側処理徹底
- **AI生成情報の誤り** → 出典明示・ユーザー注意喚起
