# 品質定義書_Vibe Walker

## 1. テスト駆動シナリオ
### シナリオ1: サクッと15分散歩
- 操作: ホーム→「サクッと15分」タップ→位置情報許可
- 期待: 5秒以内に周回ルートが地図表示、所要時間13〜17分、見どころ1件以上

### シナリオ2: しっかり60分散歩
- 操作: ホーム→「しっかり60分」タップ
- 期待: 周回距離4〜5km、緑地・住宅街優先、見どころ3件以上

### シナリオ3: 別ルート再生成
- 操作: マップ画面で「別の提案」タップ
- 期待: 前回と異なるルートが3秒以内に表示

### シナリオ4: 位置情報拒否
- 操作: 位置情報許可をキャンセル
- 期待: 「位置情報が必要です」と分かりやすく案内、住所手入力フォールバック提供

### シナリオ5: API障害
- 操作: 外部API応答なし
- 期待: 5秒でタイムアウト、リトライ案内表示、サーバーエラー記録

## 2. チェックリストと配点（合計100点）
### 機能 (35点)
- [ ] 時間選択UI動作 (5)
- [ ] 現在地取得 (5)
- [ ] ルート生成精度（±10%以内） (10)
- [ ] AIによる気持ちよさ評価反映 (8)
- [ ] 見どころ紹介表示 (7)

### 性能 (15点)
- [ ] 初回応答5秒以内 (8)
- [ ] 再生成3秒以内 (4)
- [ ] スマホでスムーズ動作 (3)

### UI/UX (20点)
- [ ] ミニマル・軽やかなデザイン (6)
- [ ] 3タップ以内で出発 (6)
- [ ] レスポンシブ対応 (4)
- [ ] アクセシビリティ AA (4)

### 品質 (10点)
- [ ] エラーハンドリング (5)
- [ ] ローディング表示 (5)

### セキュリティ (10点)
- [ ] HTTPS通信 (3)
- [ ] APIキー保護 (4)
- [ ] レート制限 (3)

### プライバシー (10点)
- [ ] 位置情報非保存 (5)
- [ ] プライバシーポリシー明示 (5)

## 3. 合否基準
- **合格**: 90点以上 かつ 必須項目すべて達成
- **条件付き合格**: 75〜89点（軽微な指摘の修正後リリース可）
- **不合格**: 74点以下、または必須項目未達
- **必須項目**: ルート生成精度／APIキー保護／HTTPS／位置情報非保存

## 4. テスト実施方法
- **ユニット**: Vitest（ルート計算・スコアリング）
- **結合**: Supertest（APIエンドポイント）
- **E2E**: Playwright（シナリオ1〜5）
- **手動**: スマホ実機（iOS/Android）チェックリスト
- **性能**: Lighthouse / 独自計測スクリプト

## 5. 品質ゲート
- **PRマージ条件**: ユニット＆結合テスト全パス、Lint通過、レビュー1名以上
- **本番デプロイ条件**: E2E全パス、性能テスト5秒以内、合計90点以上、必須項目達成
- **ロールバック条件**: エラー率1%超 または 応答10秒超
