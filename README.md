# Doraemon App — 5つの定義書 自動生成ショーケース

ざっくりしたアプリのアイデアから、開発着手に必要な5種類の定義書（**要求／要件／技術／工程／品質**）を一気通貫で自動生成する仕組みのショーケースリポジトリです。

このリポジトリには、実際にこの仕組みで生成された **架空アプリ「Vibe Walker」** の5定義書サンプルを掲載しています。

## 「Vibe Walker」とは

「今から30分だけ歩きたい」「1時間しっかり散歩したい」という気分の時に、現在地からちょうどその時間で帰ってこれる魅力的な散歩コースを提案してくれるWebアプリのアイデア。Google AI Studio × Cloud Run で構築する想定です。

このアイデアを **数行のテキスト** で伝えるだけで、以下の5つの定義書がすべて自動生成されました（生成時間: 約2分）。

## 生成されたドキュメント

| # | ドキュメント | 内容 |
|---|---|---|
| 1 | [要求定義書](./要求定義書_Vibe%20Walker.md) | アプリ概要・ターゲット・コア要求・UI/UX要求・スコープ |
| 2 | [要件定義書](./要件定義書_Vibe%20Walker.md) | 画面構成・機能詳細・非機能要件・受け入れ基準・リスクと対策 |
| 3 | [技術定義書](./技術定義書_Vibe%20Walker.md) | アーキテクチャ・データの流れ・想定コスト（非エンジニア向け平易な解説） |
| 4 | [工程定義書](./工程定義書_Vibe%20Walker.md) | Google AI Studio で組み上げるためのステップバイステップ To-Do |
| 5 | [品質定義書](./品質定義書_Vibe%20Walker.md) | テストシナリオ・チェックリスト・配点・合否基準・品質ゲート |

## 何がすごいか

- **インプットは数行のアイデア**だけ。ヒアリングシートや要件定義書のテンプレ埋めは不要
- **5ドキュメントが整合**している（要求→要件→技術→工程→品質と一貫した流れ）
- **非エンジニアでも読める**技術定義書（API＝「窓口」、コンテナ＝「箱詰め」など平易な比喩）
- **そのまま開発着手できる**工程定義書（チェックボックス形式の To-Do リスト）
- **テスト駆動の品質定義書**で、合否基準と品質ゲートまで明文化

## 想定ユースケース

- 非エンジニアの起業家が、エンジニアに発注する前にアイデアを構造化したい
- 社内DX担当が、業務アプリのアイデアを上申用ドキュメントに落としたい
- スタートアップが、AI駆動開発（Google AI Studio など）で MVP を高速立ち上げしたい
- コンサルタントが、クライアントのアイデアをその場で定義書化したい
