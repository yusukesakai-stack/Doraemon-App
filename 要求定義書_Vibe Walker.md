# 要求定義書_Vibe Walker

## 1. アプリ概要
「Vibe Walker」は、ユーザーが希望する散歩時間（例: 15分／30分／60分）を選ぶだけで、現在地からちょうどその時間で一周して戻ってこれる魅力的な散歩コースを提案するWebアプリ。マンネリ化した散歩習慣に新しい発見と気分転換を提供する。

## 2. ターゲットユーザー
- 健康・気分転換のために散歩したいが、毎回同じ道で飽きている人
- ルートを考えるのが面倒で外出が億劫になりがちな人
- 街歩きや小さな発見を楽しみたい都市生活者

## 3. コア要求
- **時間ベース**でルートを生成（往路復路含めて指定時間内に帰着）
- **気分の良い道**（緑が多い・静かな住宅街など）を優先
- **見どころ情報**を添え、散歩のモチベーションを高める

## 4. UI/UX要求
- お気に入りのスニーカーで出かけたくなる、軽やか・ミニマルなデザイン
- 開いてすぐ気分（サクッと／しっかり等）をタップ→即マップ表示
- 直感的・最小タップ・余白を活かしたレイアウト

## 5. 機能要求一覧
1. 散歩時間選択UI（15/30/60分・カスタム）
2. 現在地取得＋ルート自動生成（周回ルート）
3. ルート品質スコアリング（緑・静けさ・安全性）
4. 見どころポイント紹介（公園・カフェ・景色など）
5. 地図表示・ルートライン描画
6. 「別の提案」ボタンで再生成

## 6. 非機能要求
- 初回ルート提案までの応答時間: 5秒以内
- スマホ最優先のレスポンシブ
- 位置情報の安全な取り扱い

## 7. 制約
- Google Maps Platform API 利用
- Google AI Studio によるWebアプリビルド
- Cloud Run デプロイ前提

## 8. スコープ外
- SNS共有・コミュニティ機能
- 健康データ連携（歩数・心拍）
- 多言語対応（初期は日本語のみ）
